#+TITLE: Caler
#+OPTIONS: toc:2 num:nil

A very fast, minimalist daily planner built around the *time-blocking*
philosophy. Single-user, no login, no backend — everything lives in your
browser's ~localStorage~ and can be exported to JSON at any time. Ships as a
static site, so it can be dropped straight onto GitHub Pages (or any static
host).

* Features

- Plan the day by hour, 0-24, side by side with the previous day for context.
  Page the focused day forward/backward with ~←~/~→~ or ~a~/~d~; beyond
  yesterday/today/tomorrow the column header just shows the date.
- On first open, pick today's active hours (e.g. ~5-21~), then rapid-fire
  entries as ~DURATION TEXT~ (e.g. ~1.5 Write report~) to fill the day without
  touching the mouse. Enter just a ~DURATION~ with no text (e.g. ~1~) to skip
  that stretch of time instead of creating a task.
- Click a task to select it and edit title, description, start/end time in a
  side panel; drag a task's bottom edge to resize it; click-drag empty
  calendar space to create a new task there.
- Delete a task by double-right-clicking it, or by selecting it and pressing
  ~x~ (with a confirmation prompt).
- Week view (Monday-start) with keyboard/arrow navigation between weeks.
  Click any date's header there to plan it just like today: it prompts for
  that day's active hours first if needed, then reuses the same sequential
  quick-add flow.
- Light/dark theme, toggleable and persisted.
- Export all data as a JSON file.
- A keyboard-first flow throughout — see [[#keyboard-shortcuts][Keyboard shortcuts]].

* Getting started

** Prerequisites

- [[https://nodejs.org][Node.js]] 20+
- [[https://pnpm.io][pnpm]] (install with ~corepack enable~ or ~npm i -g pnpm~)

** Install and run

#+begin_src sh
pnpm install
pnpm dev
#+end_src

Then open the printed local URL (e.g. http://localhost:5173) in your browser.

** Other commands

| Command        | Purpose                                           |
|----------------+----------------------------------------------------|
| ~pnpm dev~     | Start the local dev server with hot reload        |
| ~pnpm build~   | Type-check and build the static site into ~dist/~ |
| ~pnpm preview~ | Serve the production build locally                |
| ~pnpm lint~    | Run oxlint over the codebase                      |

** Deploying to GitHub Pages

~vite.config.ts~ sets ~base: './'~, so the ~dist/~ output is fully relocatable
— it works unmodified on GitHub Pages, any sub-path, or even opened directly
from disk. A ready-made workflow at ~.github/workflows/deploy.yml~ builds and
publishes ~dist/~ to GitHub Pages on every push to ~main~ (enable Pages →
"GitHub Actions" as the source once, in the repo settings).

* Keyboard shortcuts
:PROPERTIES:
:CUSTOM_ID: keyboard-shortcuts
:END:

| Key                      | Action                                                                                                                                              |
|---------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------|
| ~↑~ / ~↓~ (or ~w~ / ~s~)  | Loop through the day's tasks, wrapping at either end. The arrow keys work even while typing in the quick-add bar — if there's unsaved text, you'll be asked before it's discarded. |
| ~←~ / ~→~ (or ~a~ / ~d~)  | Page one day back/forward in Day view, or one week back/forward in Week view                                                                      |
| ~.~                       | Jump the week view back to the current week                                                                                                       |
| ~q~                       | Switch to Week view (resets quick-add planning back to today)                                                                                     |
| ~e~                       | Switch to Day view (resets quick-add planning back to today)                                                                                      |
| ~i~                       | Activate the quick-add bar for whichever date is currently being planned (same as "+Continue")                                                    |
| ~x~                       | Delete the selected task (asks for confirmation)                                                                                                  |
| double-right-click        | Delete a task instantly, no confirmation                                                                                                           |
| ~t~                       | Toggle light/dark theme                                                                                                                            |
| ~Escape~                  | Close the sidebar, or cancel whichever dialog is open                                                                                             |
| ~Enter~                   | Confirm the range/quick-add form, or the focused dialog's default action                                                                          |

* Project structure

src/

├── App.tsx               Top-level state: view mode, selection, keyboard flow
├── types/index.ts         Shared domain types: Task, DayRange, Theme, ...
├── store/
│   └── plannerStore.ts    Zustand store (ranges + tasks), persisted to localStorage
├── lib/                   Framework-free helpers, all unit-testable in isolation
│   ├── constants.ts       Grid sizing, snap increment, storage key
│   ├── date.ts            Date-key <-> Date conversions, formatting, week math
│   ├── parseEntry.ts      Parses "DURATION TEXT" quick-add syntax (bare DURATION = skip)
│   ├── exportData.ts      Serializes the store to a downloadable JSON file
│   └── id.ts              UUID generation for new tasks
├── hooks/
│   ├── useTheme.ts             Syncs the store's theme with the <html> class
│   └── useDoubleRightClick.ts  Detects two right-clicks within a short window
├── components/
│   ├── Header.tsx          View switcher, week navigation, export, theme toggle
│   ├── RangeModal.tsx       First-open prompt for today's active hour range
│   ├── QuickAddBar.tsx      Sequential "DURATION TEXT" entry, auto-refocusing (bare DURATION skips)
│   ├── CalendarGrid.tsx     Shared hour-ruler + column layout (day or week)
│   ├── CalendarColumn.tsx   One day's column: grid lines, drag-to-create, tasks
│   ├── TaskBlock.tsx        A single task: resize handle, delete-on-dbl-right-click
│   ├── TaskSidebar.tsx      Edit title/description/start/end for the selected task
│   ├── ConfirmDialog.tsx    Generic Enter/Escape confirmation modal
│   └── ThemeToggle.tsx      Light/dark switch
└── styles/global.css      Tailwind entry point + base theme
#+end_src

* Stack

- [[https://vite.dev][Vite]] + React 19 + TypeScript
- [[https://tailwindcss.com][Tailwind CSS v4]] for styling (dark mode via a toggled ~dark~ class)
- [[https://github.com/pmndrs/zustand][Zustand]] with the ~persist~ middleware for local, durable state
- [[https://date-fns.org][date-fns]] for date math
- [[https://oxc.rs/docs/guide/usage/linter][oxlint]] for linting
